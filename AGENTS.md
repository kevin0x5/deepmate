# Deepmate Agent 协作规则

本文件记录 Deepmate 仓库内给 coding agent 使用的协作约束，不定义 Deepmate 的身份、人格或用户画像。

- 遵守当前仓库已有结构、命名和实现风格。
- 优先使用小而明确的改动，避免过早抽象。
- 不创建投机性的目录、空模块或占位文件。
- 保留用户已有改动，不回滚无关文件。
- 本地 Python 命令统一使用 `python3`。
- 修改代码时，如对应模块说明或实现日志存在，同步维护；不要为投机性内容新建占位文档。
- 不把 API key、密钥、原始 trace、私密用户数据写入仓库文件。
- profile memory 视为私有上下文；只有模板化、去个人化后的内容才适合提交。
